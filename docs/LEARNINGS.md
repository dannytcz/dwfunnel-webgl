# What we learned (scroll build → WebGL)

## Why Acts 1–4 felt broken

The frame-scrub videos were **technically working** but **narratively disconnected**.

| Copy said | Video showed |
|-----------|--------------|
| "Why most pages fail" | Abstract grey wireframe morph |
| "Built for your offer" | Blueprint lines filling in |
| "Behind the clicks" | Panels snapping together |

There was **no synergistic effect**: the visitor reads one story in text and sees a unrelated AI UI animation. Motion did not *prove* or *embody* the headline. It was decoration — hence "C4 to kill a chicken" and no wow.

## What works instead

1. **Motion must match meaning** — if copy is about trust, show proof; if about transformation, show before/after the *visitor* understands.
2. **One hero moment** beats five mediocre morphs — WebGL cyborg + mouse = interactive *character*, not stock abstract UI.
3. **Native resolution** — real-time render scales to the screen; 720p frame scrub never will.
4. **Sections for conversion** — problem cards, comparison, form don't need video; they need layout + copy.

## Rule for next build

> Before adding any motion, ask: *"Does this make the headline more true?"*  
> If no, cut it or replace with static.
